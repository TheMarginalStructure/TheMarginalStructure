- --- name: archive-structure-v4 description: 档案编码v4.1规则和目录结构 type: project scope: project created: 2026-06-02 priority: high --- # 档案编码规则 (PRT-0001 v4.1) ## 编码格式 `PREFIX-CODE`，最多两个连字符，长度≤20字符。 | 档案类型 | 编码前缀 | 格式 | 示例 | |---------|---------|------|------| | 阈界档案 | TMS | TMS-类型代码+4位序号 | TMS-L0234 | | 对象档案 | OBJ | OBJ-阈界代码-序号 | OBJ-O0442-01 | | 勘探记录 | EXP | EXP-阈界代码-次数 | EXP-L0234 | | 事件报告 | EVT-INC | EVT-阈界代码-INC | EVT-P0990-INC | | 通信记录 | EVT-COM | EVT-阈界代码-COM | EVT-O2847-COM | | 医疗报告 | MED | MED-阈界代码 | MED-L0734 | | 理论文件 | THY | THY-阈界代码 | THY-O0881 | | 实验记录 | EL | EL-阈界代码-V次数 | EL-O0442-V2 | | **人事档案** | **HR** | **HR-部门代码(2位)+工号(3位) = 共5位数字** | **HR-40001** | | 协议手册 | PRT | PRT-4位序号 | PRT-0001 | ## 目录结构 `content/档案/` 扁平前缀目录: - `TMS/` (阈界档案) — 文件名格式: `TMS-编码 - 中文名称.md` - `EXP/` (勘探记录 + 实验记录) - `EVT/` (事件报告 + 通信记录) - `HR/` (人事档案) - `MED/` (医疗报告) - `THY/` (理论文件) - `PRT/` (协议手册) - `OBJ/` (对象档案) - 根目录: 档案归档状态列表.md, 边际结构组织架构表.md, 人事记录表.md ## 归档状态 (5种) - 活跃 (Active) — 阈界类档案 - 归档 (Archived) — 非阈界类档案 - 封存 (Sealing) — 任务已完成/威胁已解除 - 销毁待定 (Pending Destruction) - 已销毁 (Destroyed) ## 部门编码 - 10: 最高指挥部, 20: 外勤行动部, 30: 档案与研究部 - 40: 医疗与心理部, 50: 安全与防护部, 60: 后勤与架构部 ## 人事编码规则（v4.1 变更） 格式 `HR-部门代码+3位工号`（共5位数字），如 `HR-40001`（戴维·卡特）。 注意：所有数据文件和硬编码中使用的 HR 编码必须为 5 位数字格式，禁止使用旧版 6 位格式。 ## 设施编码格式 `Site-部门编号+设施编号` (如 Site-5001 = 安全与防护部总部)
- --- name: db-update-tool-rule description: 数据库更新规则：定点修改用 db-update.ts，表结构变更用 seed type: project scope: project created: 2026-06-02 priority: high --- # 数据库更新规则 ## 两类变更分别使用不同工具 ### 1. 更新已有数据（不改表结构） **工具**: `backend/scripts/db-update.ts` **规则**: - 必须使用 `where` 条件精确锁定目标记录 - 只更新指定字段，禁止 `spread` 整个 `body` - 更新完成后打印变更摘要供人工核对 - 如果涉及的数据量超过 N 条，必须先 `count` 预览再执行 - 禁止 `updateMany` 不带 `where` 条件 ### 2. 表结构变更（新增/修改字段） **工具**: 现有 seed (`npm run db:seed`) **规则**: - seed 使用 `skipDuplicates: true`（部门）和 `findUnique → update: {}`（人员、档案） - **不会修改任何已有数据** - 只创建不存在的记录 ### 违背示例（不允许） ```typescript // ❌ 错误：spread 整个 body，可能误改非目标字段 await prisma.department.update({ where: { id: dept.id }, data: req.body, }) // ❌ 错误：无 where 条件的批量更新 await prisma.archive.updateMany({ data: { imagePath: '/api/v1/uploads/default.png' }, }) ```
